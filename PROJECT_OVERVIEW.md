# StudyRanGraph 프로젝트 개요

## 프로젝트 목적

로그 데이터를 다중 에이전트 구조로 분석하여 현재 이슈를 파악하고, 문제점을 발견하며, 분석 보고서를 자동으로 생성하는 시스템 개발

## 핵심 기술 스택

- **LangGraph**: 다중 에이전트 오케스트레이션
- **Chainlit**: 사용자 인터페이스 및 인터랙션
- **로그 소스**: MNU OUTSIDE API 서버 로그

## 프로젝트 배경

이 프로젝트는 실습 목적으로 진행되며, 실제 외부 로그 데이터를 사용하는 대신 기존에 구축된 MNU OUTSIDE API 서버에서 발생할 수 있는 로그 패턴을 기반으로 시나리오를 작성하여 진행합니다.

## 구현 방식 결정

### 검토된 두 가지 접근 방식

#### 1. 정적 시나리오 & 데이터셋 방식 ✅ (선택됨)
- 로그 내용을 분석하여 시나리오 작성
- 시나리오별 다중 데이터셋 생성
- 각 시나리오에 맞춰 테스트 수행

**장점:**
- 구현 복잡도가 낮음
- 테스트 및 검증이 용이함
- 재현 가능한 시나리오로 안정적인 개발 가능

#### 2. 실시간 로그 생성기 방식 ❌ (보류)
- 로그 스키마 및 예제 데이터셋 기반
- 실시간으로 로그를 생성하는 생성기 개발
- 동적인 로그 패턴 시뮬레이션

**보류 이유:**
- 개발 복잡성이 높음
- 초기 실습 프로젝트로는 과도한 범위

### 최종 결정: 정적 시나리오 & 데이터셋 방식

개발 복잡성을 고려하여 시나리오 기반 접근 방식을 선택하였습니다. 이 방식은 명확한 테스트 케이스와 예측 가능한 결과를 제공하며, 다중 에이전트 시스템의 핵심 기능 구현에 집중할 수 있게 합니다.

## 프로젝트 구조 계획

### 1. 시나리오 설계 단계
- MNU OUTSIDE API 서버의 일반적인 로그 패턴 분석
- 다양한 이슈 시나리오 정의 (에러, 성능 문제, 보안 이슈 등)
- 각 시나리오별 특징 및 기대 분석 결과 명세

### 2. 데이터셋 생성 단계
- 각 시나리오에 대응하는 로그 데이터셋 작성
- 정상 로그와 비정상 로그의 혼합
- 다양한 엣지 케이스 포함

### 3. 에이전트 시스템 구현 단계
- LangGraph를 활용한 다중 에이전트 구조 설계
- 로그 분석, 이슈 탐지, 보고서 생성 에이전트 구현
- 에이전트 간 협업 및 데이터 흐름 정의

### 4. UI/UX 구현 단계
- Chainlit을 활용한 인터랙티브 인터페이스
- 로그 업로드 및 분석 결과 시각화
- 실시간 분석 진행 상황 표시

## 다음 단계

- 효과적인 로그 분석 워크플로우 구축
